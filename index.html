üöÄ Publish / Deploy

Important (read this):

This app stores users + passwords in localStorage (browser only). If you publish it, every visitor will have their own separate user database on their own device. For a real production login system, you need a backend + database.

Option A ‚Äî GitHub Pages (free + easiest)

  1. Create a GitHub repo (example: login-system-pro).
  2. Upload index.html, styles.css, app.js to the repo.
  3. Go to Settings ‚Üí Pages.
  4. Choose Deploy from a branch ‚Üí branch main ‚Üí folder /root.
  5. Wait ~1 minute ‚Üí you get a URL like https://yourname.github.io/login-system-pro.

Option B ‚Äî Netlify (free + great for custom domains)

  1. Go to netlify.com ‚Üí sign in.
  2. Drag-and-drop your project folder OR connect your GitHub repo.
  3. Build settings: none needed (static site).
  4. Netlify gives you a URL like https://something.netlify.app.
  5. To use www.yourdomain.com: Site settings ‚Üí Domain management ‚Üí Add domain.

Option C ‚Äî Vercel / Cloudflare Pages

Both are similar: import the GitHub repo ‚Üí deploy as a static site ‚Üí then add a custom domain in the dashboard.

Custom Domain (like www.login.com)

1) You must buy the domain first (Namecheap, GoDaddy, Cloudflare Registrar, etc.).

2) In your hosting dashboard (Netlify/Vercel/Cloudflare/GitHub Pages), add the domain.

3) Update your domain DNS:

  • www usually becomes a CNAME to your host (Netlify/Vercel gives you the exact value).
  • The root domain login.com is usually an A record or ALIAS/ANAME depending on provider.

4) Turn on HTTPS (most hosts do it automatically with Let‚Äôs Encrypt).
